Model

The graph model supported by the framework generalizes the core RDF model in the following respects:

  • Every term has a lexical form and a datatype: blank nodes, URI references and plain literals are modeled as typed literals with an implementation-specific internal datatype and a lexical form corresponding respectively to the internal node identifier, the referenced URI and the textual content.
  • Blank nodes and literals are allowed as triple subjects and predicates. Encoders are responsible for reporting or filtering generalized triples during serialization to formats that don't support them.

Datalog

The framework supports a Datalog based integrated language providing uniform query, inference and update functions over generalized RDF graphs.

Datalog programs are conveniently written in a Turtle-based language. A dedicated decoder translates this human-readable syntax into an equivalent machine-readable description based on a RDF vocabulary, which might also be directly used to build programs programmatically. The execution engine compiles machine-readable models into collections of rules and queries, which are a finally applied to a target graph by a forward-chaining evaluator.

Function Calls

The framework provides a standard runtime library of functions ad operators.

Error handling

Evaluation errors are signaled by returning a null value. With the exception of binary logical operators and the bound guard predicate, functions and operators included in the runtime library raise an error (i.e. return null) if any of their arguments is null (i.e. is the result of a faulty expression).

Type promotions

Arguments passed to functions and operators included in the runtime library are promoted to the expected type: an error is raised if any of an argument is not derived from or can't be promoted to the expected type. Promotions are uniformly handled through the API of the Type class allowing custom datatypes to behave on a par with native ones.

Reasoners

Reasoners are specialized Datalog programs implementing specific graph entailment regimes. The framework provides them either as Datalog source files or as Datalog graphs describing clauses using the Datalog RDF vocabulary. Source files are included in the distribution as resources and may be accessed using the special java: URI scheme. Graphs are lazily decoded from source files and made available through utility methods on namespace classes.

OWL RL

The OWL RL reasoner implements the RL/RDF rules axiomatization of the OWL 2 knowledge representation language, with the following known deviations:

  • no support for reasoning over typed literals (rules dt-type1, dt-type2, dt-eq, dt-diff, dt-not-type);
  • no support for reasoning over annotation properties (rule prp-ap);
  • exclusion of reflexive statements (rules eq-ref, scm-cls, scm-op, scm-dp) and of statements about owl:Nothing (rule scm-cls).

Typed literals and annotation properties are not supported for performance reasons, while reflexive statements and statements about owl:Nothing are excluded to produce more readable results.

OWL Base

The OWL Base reasoner implements a basic axiomatization limited to the materialization of class/property hierarchies (rdfs:subClassOf, rdfs:subPropertyOf), transitive (owl:TransitiveProperty) and symmetric properties (owl:SymmetricProperty), rdf:type and owl:sameAs.
