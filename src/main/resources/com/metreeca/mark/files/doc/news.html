2011.03.10

Version 0.6 Released

This version refactors the abstract I/O framework and extends it with generalized support for decoding from XML transform sources and encoding to XML transform results. It also addresses residual conformance issues in the core RDF model and in URI-handling classes. A few incompatible changes were introduced in the API, most notably the net.oroboro.ttl package was renamed to net.oroboro.txt to reflect its actual content. Other noteworthy changes include:

Model
  • Cleaned up and extended Node constructors and accessors.
  • Upgraded blank node identifiers to UUIDs.
  • Implemented strict RDF graph view.
  • Downgraded the return value of Graph.size() to an estimate.
  • Fixed canonicalization of RDF URI references.
  • Fixed Unicode normalization of RDF literals.
Catalog
  • Decoupled URI qualification, compaction and formatting in Catalog.
  • Added support for (safe) CURIEs in Catalog.compact()/.expand().
Codecs
  • Made encoders ignore generalized edges.
  • Migrated TTL/RDFEncoder to breadth-first subjects ordering.
Datalog
  • Added support for single quote strings.
  • Implemented item concatenation in sequence expressions.
  • Extended the runtime library with blank node constructors and additional XPath 2.0 functions (boolean, lang, uri-to-iri, iri-to-uri, escape-html-uri, exists, empty, distinct-values).
  • Fixed sequence to atomic unwrapping: now rejects non-singleton sequences.
  • Fixed XPath logical operators: now correctly handle sequence operands.
  • Fixed XSDString.promote(): now accepts tagged literals.

2011.02.10

Version 0.5 Released

This release expands and consolidates the runtime support for core XSD datatypes and related XPath 2.0 functions and operators. XSD constructor, casting and promotion methods were reviewed and completed. XPath functions for numerics and strings were implemented and aggregate functions were aligned to the XPath specs. The reference documentation was reorganized and extended with a quick reference for the runtime library. Other noteworthy changes include:

  • Made all XSD-related classes stictfp.
  • Added support for constructors, casts and promotions for custom datatypes.
  • Simplified the syntax for invoking aggregate functions in Datalog.
  • Added support for comments in JSONReader.

2010.12.07

Version 0.4 Released

This release consolidates the Datalog language: Datalog and Turtle decoders were decoupled and the Datalog grammar overhauled and simplified. The tutorial was aligned to the updated grammar, which introduces some incompatible changes, to make naming and precedence rules more consistent: most notably, the select clause and sequence modifiers were renamed and clause labeling redefined to avoid ambiguities.

The package hierarchy was also simplified by merging the Datalog support package into the core model and reorganizing I/O adapters into four packages dedicated respectively to SAX-based transcribers, generic, Turtle and XML-based codecs.

Other noteworthy changes include:

  • Implemented expand and create queries (akin to SPARQL's describe and construct).
  • Implemented the exists connective (akin to SPARQL's pattern)
  • Extended the XPath library implementation.
  • Fixed handling of unbound named variables in templates.

2010.11.17

Version 0.3 Released

This release introduces Datalog-based OWL reasoning support: the OWL RL reasoner implements the full OWL 2 RL/RDF rules axiomatization, while the OWL Base reasoner implements a basic axiomatization limited to class/property hierarchies, transitive and symmetric properties and equality. Other noteworthy changes include:

  • Added graph updating to Engine.select()/verify().
  • Implemented Morpher.verify().
  • Added support for multi-origin programs in Morpher.select()/update().
  • Added output flushing to encoders.
  • Fixed Query.subjects().
  • Fixed handling of lists in Datalog clause templates.
  • Fixed handling of URI references in lists in Turtle and RDF encoders.
  • Fixed handling of default namespaces in RDF encoder.

2010.10.29

Version 0.2 Released

This release improves the usability of the command-line shell and extends its features. Constraints verification is now supported and the conversion/scripting mode is fully functional and documented. Other noteworthy changes include:

  • Added content-based format-guessing to AnyDecoder.
  • Improved catalog sharing and merging: default and duplicated namespace prefixes are now handled more intuitively.
  • Implemented Datalog conditional expressions and substring functions.
  • Laid down support for custom type promotion and casting.

2010.10.11

Version 0.1 Released

This is the initial public release. The code is fairly stable and reasonably tested, even if some changes in the package structure and in the API should be expected until the first release in the 1.x series. The Datalog engine is being actively improved, with the goal of formally supporting OWL RL reasoning in the upcoming releases.
