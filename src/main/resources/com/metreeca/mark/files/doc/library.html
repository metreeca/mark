Introduction

This reference provides a concise overview of datatypes, functions and operators included in the runtime library for the Datalog based language supported by the Oroboro Java RDF processing framework.

Datatypes

The runtime library provides native support the following datatypes:

xsd:double

In the following sections the term atomic denotes any atomic value, including blank nodes, URI references and plain literals. The term numeric denotes the types xsd:integer, xsd:decimal, xsd:float, and xsd:double. Functions and operator whose operands and result are designated as atomic or numeric might be thought of as representing a collection of items, one for each of the matched types.

Terms in the form type?, type*, type+ denote sequences including respectively zero or one, zero or more and one or more atomic values of the specified type.

The result of this promotion is created by casting the original value to the required type. This kind of promotion may cause loss of precision.

xsd:anyURI

xsd:string

The result of this promotion is created by casting the original value to the type xsd:string. Since xsd:anyURI values can be promoted to xsd:string, functions and operators that compare strings also compare xsd:anyURI. This ensures that orderings that include strings, xsd:anyURI values, or any combination of the two types are consistent and well-defined. 

URI Reference

xsd:anyURI

The result is the xsd:anyURI value that has the same lexical form as the original value. Since xsd:anyURI values can be promoted to xsd:string, functions and operators that handle strings also handle URI references.

Plain Literal

xsd:string

The result is the xsd:string value that has the same lexical form as the original value, stripped of any language tag.

Native Functions

Constructors

blank() as blank

Returns a unique blank node.

blank($nodes as atomic*) as blank

Returns a blank node uniquely identifying the nodes included in the specified sequence, regardless of their ordering and ignoring null items.

This constructor is idempotent and returns the same blank node when invoked on sequences containing the same set of non-null nodes.

reference($uri as xsd:string) as reference

Returns a reference to the specified uri.

An error is raised if uri is malformed or not absolute.

literal($lexical as xsd:string) as simple literal

Returns a simple literal with the specified lexical form.

literal($lexical as string, $tag as xsd:string) as tagged literal

Returns a tagged literal with the specified lexical form and language tag.

An error is raised if tag is not a legal language tag.

literal($lexical as string, $uri as xsd:anyURI) as typed literal

Returns a typed literal with the specified lexical form and datatype uri. Note that typed literals may also be built using specific type constructor functions identified by the URI of the datatype.

An error is raised if uri is malformed or not absolute or if the lexical form doesn't belong to the lexical space of the datatype.

Guard Predicates

is-bound($node as atomic) as xsd:boolean

Returns true if the argument is actually bound to a value; false otherwise.

is-blank($node as atomic) as xsd:boolean

Returns true if node is a blank node; false otherwise.

is-reference($node as atomic) as xsd:boolean

Returns true if node is a URI reference; false otherwise.

is-literal($node as atomic) as xsd:boolean

Returns true if node is either a plain or a typed literal; false otherwise.

is-plain($node as atomic) as xsd:boolean

Returns true if node is a plain literal, possibly with a language tag; false otherwise.

is-simple($node as atomic) as xsd:boolean

Returns true if node is a plain literal with no language tag; false otherwise.

is-tagged($node as atomic) as xsd:boolean

Returns true if node is a plain literal with a language tag; false otherwise.

is-tagged($node as atomic, $tag-range as xsd:string) as xsd:boolean

Returns true if node is a plain literal with a language tag matching the specified basic language range; false otherwise.

is-typed($node as atomic) as xsd:boolean

Returns true if node is a typed literal; false otherwise.

is-typed($node as atomic, $type as xsd:anyURI) as xsd:boolean

Returns true if node is a typed literal with the specified type; false otherwise.

An error is raised if type is not an absolute URI.

is-numeric($node as atomic) as xsd:boolean

Returns true if node is a typed literal with a datatype equal to or derived from xsd:integer, xsd:decimal, xsd:float, or xsd:double.; false otherwise.

is-nan($node as numeric) as xsd:boolean

Returns true if node is an xsd:double or xsd:float representing a Not-a-Number (NaN) value; false otherwise.

is-infinite($node as numeric) as xsd:boolean

Returns true if node is an xsd:double or xsd:float representing an infinity; false otherwise.

Accessors

text($node as atomic) as simple literal

Returns the lexical form of node as a simple literal.

type($node as atomic) as reference

Returns a reference to the URI of the datatype of node.

lang($node as atomic) as simple literal

Returns the language tag of node as a simple literal.

An error is raised if node is not a tagged plain literal.

value($node as atomic) as atomic

Returns the canonical form of node.

An error is raised if the lexical form of node doesn't belong to the lexical space of its datatype.

order($node as atomic, $reference as atomic) as xsd:integer

Returns -1, 0, or 1, depending on whether $node is respectively lexically less than, equal to, or greater than $reference.

XPath 2.0 Functions

The runtime library implements a subset of the functions defined by XQuery 1.0 and XPath 2.0 Functions and Operators (Second Edition) under the following restrictions:

  • The framework supports only atomic values and sequences of atomic values: references to nodes and sequences of nodes are ignored.
  • Atomic values and sequences are considered as disjoint spaces: functions and operators handling atomic values don't accept empty sequences.
  • Only the default Unicode Codepoint Collation is supported.

5 Constructor Functions

5.1 Constructor Functions for XML Schema Built-in Types 

Every built-in XSD datatype has an associated constructor function named after its URI according to the following schema:

xsd:type($arg1 as atomic) as xsd:type

5.4 Constructor Functions for User-Defined Types 

Every custom datatype plugged into the global type registry has an associated constructor function named after its URI according to the following schema:

<type>($arg1 as atomic) as <type>

Extended Functions on Strings Values

fn:string-join($arg as xsd:string*) as xsd:string

Returns an xsd:string created by concatenating the members of $arg.

This version of the function is equivalent to fn:string-join($arg as xsd:string*, $separator as xsd:string) called with $separator set to the empty string.

fn:uri-to-iri($uri as xsd:string) as xsd:string

Returns an xsd:string in Unicode Normal Form C representing the IRI equivalent to the specified uri.

An error is raised if uri doesn't represent a legal URI.

Extended Functions that Use Pattern Matching

fn:tokenize($text as xsd:string) as xsd:string+

Returns a sequence of one or more xsd:string whose values are substrings of text separated by one or more whitespace characters. The separators themselves are not returned.

This version of the function is equivalent to fn:tokenize($input as xsd:string, $pattern as xsd:string) called with $pattern set to "\\s+".

Extended Aggregate Functions 

fn:max($x as atomic, $y as atomic) as atomic

Returns $x if its value is greater than or equal to the value of $y; $y otherwise.

This version of the function is equivalent to fn:max($arg as atomic*) called with $arg set to [$x, $y].

fn:min($x as atomic, $y as atomic) as atomic

Returns $x if its value is less than or equal to the value of $y; $y otherwise.

This version of the function is equivalent to fn:min($arg as atomic*) called with $arg set to [$x, $y].
